Технические различия словами
1. Структура данных
Мой код: Использует простой двумерный массив для хранения имён файлов

Код ИИ: Создаёт структуру FileList, которая объединяет массив файлов и счётчик в одном объекте

2. Управление памятью
Мой код: Фиксированный размер массива (100×100), нет проверки переполнения

Код ИИ: Проверяет не превысили ли лимит файлов, защита от выхода за границы массива

3. Безопасность работы со строками
Мой код: Использует sprintf без проверки длины (может переполнить буфер)

Код ИИ: Использует snprintf с указанием максимальной длины (безопаснее)

4. Фильтрация результатов поиска
Мой код: Выводит всё что найдёт, включая папки если они попадут под маску

Код ИИ: Проверяет атрибуты файла и пропускает папки (флаг _A_SUBDIR)

5. Инкапсуляция логики
Мой код: Логика проверки уникальности в отдельной функции, но работа с массивом напрямую

Код ИИ: Операции с файлами инкапсулированы в функции работы со структурой

6. Обработка граничных случаев
Мой код: Базовая обработка (файлов нет — пропустить)

Код ИИ: Полная обработка (длинные пути, переполнение, неверные маски)

7. Подход к именованию
Мой код: Короткие имена (cnt, has, el), смесь языков

Код ИИ: Полные имена (file_count, file_exists), только английский

8. Разделение ответственности
Мой код: Основная логика в main(), вспомогательные функции минимальны

Код ИИ: Чёткое разделение: инициализация, проверка, добавление, вывод

9. Работа с путями
Мой код: Использует / (работает в Windows, но нестандартно)

Код ИИ: Использует \\ (стандартный разделитель для Windows)
